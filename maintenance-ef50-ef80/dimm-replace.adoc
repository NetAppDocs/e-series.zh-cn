= 更换 DIMM - EF50 和 EF80
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
更换 EF50 或 EF80 存储系统中的 DIMM，如果该 DIMM 出现故障、未处于最佳状态或容量不匹配。

.关于此任务
* 通过以下方式更换 DIMM：确定是否需要更换，准备，将受损控制器脱机，卸下控制器，更换 DIMM，重新安装控制器，将其置于联机状态，完成更换，并将故障部件退回 NetApp。
* 在拔下受损控制器的任何电缆之前，必须将其置于离线状态。
* 您可以打开存储系统位置（蓝色）LED（位于存储系统和两个控制器的正面），以帮助物理定位受影响的存储系统。使用 SANtricity System Manager，选择*硬件* > *控制器和组件*，选择*控制器架*选项卡，然后从上下文菜单中选择*打开定位灯*。




== 第 1 步：确定是否需要更换 DIMM

出现故障、报告未处于最佳状态或容量不匹配时，请更换 DIMM。

.步骤
. 在 SANtricity System Manager 中，访问受损控制器的存储控制器配置文件：
+
.. 转至 *Support* > *Support Center*。
.. 在支持资源页面中，选择 *Storage Array Profile*。
.. 向下滚动或使用搜索字段查找 * 数据缓存模块 * 信息。


. 如果 DIMM 显示以下任一条件，请记下 DIMM 的位置并继续此过程以更换 DIMM。
+
如果这些条件都不存在，则无需更换 DIMM。退出此过程。

+
** 出现故障的 DIMM
** DIMM 将 *Data Cache Module* 报告为非最佳
** *Data Cache Module* 容量不匹配的 DIMM






== 第2步：准备更换DIMM

准备更换 DIMM 时，请确保没有其他需要解决的存储系统问题，确保卷处于正确状态，收集必要的工具和设备，采取预防措施以防止静电放电，并备份存储系统的配置数据库。

.步骤
. 解决任何其他存储系统问题。
+
使用 SANtricity System Manager，查看 Recovery Guru 中的详细信息，以查看是否存在任何其他问题。

. 请确保未使用任何卷，或者在使用这些卷的所有主机上安装了多路径驱动程序。
. 确保您已具备以下条件：
+
** 适合您的存储系统配置的正确大小的替换 DIMM。
** 防静电腕带或您已采取其他防静电预防措施。
** 平坦、无静电的工作表面。
** 用于标识连接到受损控制器的每根电缆的标签。
** 具有浏览器的管理站，可以访问受损控制器的 SANtricity System Manager。
+
要打开 System Manager 界面，请将浏览器指向受损控制器的域名或 IP 地址。



. 在处理和安装 DIMM 期间防止静电放电（ESD）：
+
** 请将 DIMM 放在 ESD 袋中，直到准备好安装为止。
** 用手打开 ESD 袋或用剪刀剪掉顶部。请勿将金属工具或刀片插入 ESD 袋中。
** 保留 ESD 袋和任何包装材料以退回故障的 DIMM，或在运送 DIMM 时使用经批准的包装。
** 始终佩戴防静电腕带，将其接地到存储系统机箱上未上漆的表面。
** 避免磁场。使 DIMM 远离磁性设备。


. 使用 SANtricity System Manager 备份存储系统的配置数据库：
+
如果删除控制器时出现问题，您可以使用保存的文件还原您的配置。系统保存 RAID 配置数据库的当前状态，其中包括受损控制器上卷组和磁盘池的所有数据。

+
.. 选择 * 支持 * > * 支持中心 * > * 诊断 * 。
.. 选择 * 收集配置数据 * 。
.. 选择 *Collect*。
+
此文件将保存在浏览器的 "Downloads" 文件夹中，名为 * configurationData-<arrayName>-<DateTime>.7z* 。







== 步骤 3：将控制器置于离线状态

将受损的控制器脱机，以便您可以安全地执行此过程的其余部分。

.关于此任务
每当您将控制器置于离线状态时，都需要等待至少一分钟才能使控制器恢复联机。此等待期允许存储系统更新控制器的状态，并确保将所有缓存数据写入驱动器。

.步骤
. 如果受损控制器尚未离线，请使用 SANtricity System Manager 将其置于离线状态：
+
.. 选择 *Hardware* > *Controllers and components* 以显示控制器。
.. 选择要置于脱机状态的控制器以显示其内容菜单。
.. 选择 *Place offline*，然后确认您要执行此操作。
+

NOTE: 如果您使用尝试脱机的控制器访问 SANtricity System Manager，则会显示 SANtricity System Manager Unavailable 消息。选择*连接到备用网络连接*以使用其他控制器自动访问 SANtricity System Manager。



. 等待 SANtricity System Manager 将控制器状态更新为脱机。
+

CAUTION: 更新状态之前，请勿开始任何其他操作。

. 使用 Recovery Guru 确认移除有故障的组件是安全的：
+
.. 选择 *Recheck*。
.. 确认详细信息区域中的 *OK to remove* 字段显示 *Yes*。
+

CAUTION: 如果 *OK to remove* 字段未显示 *Yes*，请不要继续移除有故障的组件。相反，请使用 Recovery Guru 对问题进行故障排除。







== 步骤 4：卸下控制器

验证缓存内存中的任何数据是否已写入驱动器，从受损控制器拔下所有电缆，然后从机箱中取出受损控制器。

.步骤
. 在受损的控制器上，确保 NV Caching Active（绿色）LED 熄灭。
+
当 NV Caching Active（绿色）LED 熄灭时，缓存存储器中的任何数据都已写入驱动器，可以安全地移除受损的控制器。

+

NOTE: 如果 NV Caching Active（绿色）LED 指示灯亮起，则缓存数据正在写入驱动器。您必须等待过程完成，并且 NV Caching Active LED 指示灯关闭。但是，如果 LED 指示灯持续亮起超过五分钟，请在继续此过程之前联系 https://mysupport.netapp.com/site/global/dashboard["NetApp 支持"]。

+
NV 缓存活动（绿色）LED 位于控制器上的 NV 图标旁边。

+
image::../media/drw_g_nvmem_led_ieops-1839.svg[NV 状态 LED 位置]

+

 a|

. 戴上 ESD 腕带或采取其他防静电预防措施。
. 标记连接到受损控制器的每根电缆。
. 从受损控制器的电源上拔下电源线。
+

NOTE: 电源 (PSU) 没有电源开关。

. 从受损控制器上拔下所有电缆。
. 卸下受损的控制器：
+
下图显示了卸下控制器时控制器手柄的操作（从控制器的左侧）：

在控制器的两端，向外推动垂直锁定凸耳，将手柄释放到水平位置。



 a| 
image::../media/icon_round_2.png[标注编号 2]
 a| 
** 将手柄拉向您，将控制器从中间背板上卸下。
+
当您拉动时，手柄从控制器伸出，然后您感到一些阻力，继续拉动。

** 将控制器滑出机箱，同时支撑控制器底部，并将其放置在平坦、无静电的工作表面上。




 a|

 a| 
如果需要，将手柄向上旋转（选项卡旁边），将它们移开。

. 逆时针旋转指旋螺钉以松开控制器盖，然后打开盖子。




== 第 5 步：更换 DIMM

在损坏的控制器中找到有故障的 DIMM 并进行更换。

.步骤
. 如果您尚未接地，请正确接地。
. 找到有故障的 DIMM。
+

NOTE: 有关确切的 DIMM 位置，请参阅 https://hwu.netapp.com["Netapp Hardware Universe"]或控制器封面上的 FRU 图。

. 卸下有故障的 DIMM：
+
image::../media/drw_g_dimm_ieops-1873.svg[DIMM 更换]

+
[cols="1,4"]
|===


 a| 
image::../media/icon_round_1.png[标注编号 1]
 a| 
DIMM 插槽编号和位置。


NOTE: 根据您的存储系统型号，您将有两个或四个 DIMM。



 a| 
image::../media/icon_round_2.png[标注编号 2]
 a| 
** 请注意插座中 DIMM 的方向，以便使用相同的方向插入更换的 DIMM。
** 通过缓慢推开 DIMM 插槽两端的两个 DIMM 弹出凸耳，弹出有故障的 DIMM。



IMPORTANT: 小心地握住 DIMM 的角落或边缘，以避免 DIMM 电路板组件受到压力。



 a| 
image::../media/icon_round_3.png[标注编号 3]
 a| 
将 DIMM 向上提起并从插槽中取出。

弹出器卡舌保持在打开位置。

|===
. 拆开替换 DIMM 的包装，并将其放置在存储系统附近的平坦无静电表面上。
+
保存用于退回故障 DIMM 的包装材料。

. 安装更换后的 DIMM：
+
.. 确保连接器上的 DIMM 弹出器卡舌处于打开位置。
.. 握住 DIMM 的四角，并将其垂直插入插槽。
+
DIMM 底部的插针之间的凹口应与插槽中的卡舌对齐。

+
正确插入时，DIMM 可以轻松插入，但在插槽中紧密贴合。如果觉得 DIMM 插入不正确，请重新插入。切勿强行将 DIMM 插入插槽。

.. 目视检查 DIMM，确保其均匀对齐并完全插入插槽。
.. 在 DIMM 的顶部边缘小心但牢固地向下推，直到弹出器卡舌在 DIMM 两端的凹口上卡入到位。






== 第 6 步：重新安装控制器

将控制器重新安装到机箱中，重新连接其电源线及其所有电缆。

.关于此任务
下图显示了重新安装控制器时控制器手柄（从控制器的左侧）的操作，可以用作控制器重新安装步骤的参考。

image::../media/drw_g_and_t_handles_reinstall_ieops-1838.svg[控制器句柄操作以安装控制器]

[cols="1,4"]
|===


 a| 
image::../media/icon_round_1.png[标注编号 1]
 a| 
如果您在维修控制器时将控制器手柄直立旋转（在选项卡旁边）以将其移开，请将其向下旋转到水平位置。



 a| 
image::../media/icon_round_2.png[标注编号 2]
 a| 
按下手柄，将控制器重新插入机箱。



 a| 
image::../media/icon_round_3.png[标注编号 3]
 a| 
将手柄旋转到直立位置，并使用锁定卡舌锁定到位。

|===
.步骤
. 合上控制器盖，顺时针旋转指旋螺钉，直至拧紧。
. 将控制器插入机箱：
+
.. 将控制器后部与机箱中的开口对齐，轻轻但牢固地按下手柄，直到控制器与中间背板相接并完全就位。
+

NOTE: 将控制器滑入机箱时不要用力过大；这可能会损坏连接器。

.. 向上旋转控制器手柄，并用卡舌锁定到位。


. 将电源线重新连接到电源，并使用电源线固定器固定电源线。
+
电源恢复供电后，状态 LED 应为绿色。

. 将所有电缆重新连接到控制器。
+

NOTE: 在将控制器置于在线状态之前，必须重新连接电缆。这对于镜像电缆连接尤其重要，因为它们可确保完整的系统冗余，并用于缓存镜像和 I/O 传输。





== 第 7 步：将控制器置于联机状态

使受损的控制器恢复联机。

.步骤
. 使用 SANtricity System Manager 将控制器置于联机状态：
+
.. 选择 *Hardware* > *Controllers and components* 以显示控制器。
.. 选择要置于联机状态的控制器以显示其上下文菜单。
.. 选择 *Place online*，然后确认您要执行此操作。
+
系统将控制器置于联机状态。



. 在控制器启动时，检查控制器 LED 。
+
重新建立与另一控制器的通信时：

+
** 琥珀色警示 LED 仍保持亮起状态。
** 主机链路 LED 可能亮起，闪烁或熄灭，具体取决于主机接口。


. 当控制器恢复联机时：
+
.. 确认其状态为最佳。
.. 确认控制器上的注意 LED 熄灭。
+
如果状态不是 Optimal 或任何 Attention LED 指示灯亮起，请确认所有电缆都已正确安装，并且控制器已正确安装。如有必要，请卸下并重新安装控制器。

+

NOTE: 如果无法解决问题，请联系 https://mysupport.netapp.com/site/global/dashboard["NetApp 支持"^]，然后再继续此过程。如果需要，请使用 SANtricity System Manager 为您的存储系统收集支持数据。







== 第 8 步：完成 DIMM 更换

使用 SANtricity System Manager，确保已安装最新版本的 SANtricity 操作系统，验证所有卷都已返回到其拥有的控制器，并收集支持数据，以便您可以恢复操作。

.步骤
. 请确保存储系统上已安装最新版本的 SANtricity 操作系统：
+
.. 选择 *Support* > *Upgrade Center*
.. 根据需要安装最新版本。


. 验证所有卷都已返回到其拥有的控制器：
+
[cols="1,2"]
|===
| 条件 | 操作 


 a| 
Recovery Guru 存在并指示卷不在其首选路径上（未返回到其所属控制器）
 a| 
.. 将卷重新分配给其拥有的控制器，通过选择 *Storage* > *Volumes* > *More* ，然后从下拉菜单中选择 *Redistribute volumes* 。
.. 通过检查 Recovery Guru 是否存在并且仍然指示卷不在其首选路径上，验证所有卷已重新分配给其拥有的控制器。



NOTE: 如果卷仍未返回到其拥有的控制器，请联系  https://mysupport.netapp.com/site/global/dashboard["NetApp 支持"]。



 a| 
Recovery Guru 不存在（似乎所有卷都已返回给其拥有的控制器）
 a| 
通过选择 *Storage* > *Volumes* > *More* 来验证所有卷是否已返回其拥有的控制器，然后从下拉菜单中选择 *Change volume ownership* 以查看卷所有者。

|===
. 为您的存储系统收集支持数据：
+
.. 选择 * 支持 * > * 支持中心 * > * 诊断 * 。
.. 选择 * 收集支持数据 * 。
.. 选择 *Collect*。
+
此文件将保存在浏览器的 "Downloads" 文件夹中，名为 * support-data.7z* 。







== 第 9 步：将故障部件退回 NetApp

如套件附带的 RMA 说明中所述，将故障部件退回 NetApp。有关更多信息，请参阅 https://mysupport.netapp.com/site/info/rma["零件退货和更换"]页面。
