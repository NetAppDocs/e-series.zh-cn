= 用缆线连接硬件 - EF50 和 EF80
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
安装 EF50 或 EF80 存储系统硬件后，连接控制器间镜像和主机网络连接。（稍后在"完成存储系统设置"部分对管理端口进行布线。）

.关于此任务
* 术语 _I/O 模块_ 用于指代本过程中的主机接口卡 (HIC)。
* 电缆图形有箭头图标，显示将连接器插入端口时电缆连接器拉片的正确方向（向上或向下）。
+
插入连接器时，您应该感觉到它咔嗒一声卡入到位；如果感觉不到咔嗒声，请将其取出，翻转后重试。

+
image:../media/drw_cable_pull_tab_direction_ieops-1699.svg["电缆拉舌方向"]





== 步骤 1：连接控制器间镜像连接线

将控制器彼此连接以启用控制器间镜像。控制器间镜像连接可确保完整的系统冗余，并用于缓存镜像和 I/O 传输。EF50 和 EF80 存储系统的布线相同。


NOTE: 无论存储系统中安装的控制器间镜像 I/O 模块的速度如何（存储系统支持 100 GbE 或 200 GbE），您都可以使用随附的 200 GbE 电缆。如果控制器间镜像 I/O 模块的速度为 100 GbE，则连接将以较低的速度（100 GbE）运行。

.步骤
. 将控制器连接在一起：
+
.. 电缆控制器 A 端口 e4a 到控制器 B 端口 e4a。
.. 电缆控制器 A 端口 e4b 到控制器 B 端口 e4b。
+
*200 GbE 以太网电缆*

+
image::../media/oie_cable100_gbe_qsfp28.png[用于镜像连接的 100 gbe 以太网电缆]

+
image:../media/drw_ef50-ef80_mirroring_2p_100gbe_ieops-2659.svg["ef50 和 ef80 控制器到控制器镜像连接电缆"]







== 步骤 2：连接主机电缆

根据您的网络拓扑（直接连接或光纤连接）为存储系统连接主机电缆。

.关于此任务
* 根据您的存储系统型号，存储系统中安装的主机 I/O 模块类型可以是以太网或光纤通道 (FC)。此过程中的布线示例显示了存储系统模型支持的两种类型的主机 I/O 模块。
* 布线示例未显示主机 1 和主机 2 中的 4 端口 64Gb FC HBA；但是，如果安装了这些 HBA，则以与 2 端口 HBA 相同的方式连接其他端口。


[role="tabbed-block"]
====
.直连式拓扑
--
以下示例显示了使用直接连接拓扑将存储系统连接到主机的布线情况。

.EF50 具有两个 4 端口 64Gb FC I/O 模块
[%collapsible]
=====
.关于此任务
* 布线示例显示了插槽 1 和 2 中的主机 I/O 模块。这是 EF50 存储系统支持的最大主机 I/O 模块数。但是，仅需要插槽 1 中的主机 I/O 模块；插槽 2 中的主机 I/O 模块是可选的。
+
如果您的存储系统安装了一个主机 I/O 模块，则可以忽略与其他主机 I/O 模块的缆线连接，而仅将缆线连接到已安装的主机 I/O 模块。

* 直连存储系统具有两条单独的冗余路径：路径 A 和路径 B。
+
** 路径 A 连接显示为主机和控制器上的蓝色电缆和蓝色端口。它将每个主机上的 HBA 端口连接到控制器 A 端口 a 和 c。
** 路径 B 连接显示为主机和控制器上的红色电缆和红色端口。它将每个主机上的 HBA 端口连接到控制器 B 端口 a 和 c。


* 虽然布线示例显示 I/O 模块端口 a 和 c 连接到主机，但您可以使用端口 a 和 b 或端口 c 和 d。


.步骤
. 用缆线将主机连接到控制器：
+
.. 将主机 1 路径 A（蓝色）HBA 端口连接到控制器 A 的 a 端口（1a 和 2a）。
.. 将主机 1 路径 B（红色）HBA 端口连接到控制器 B 的 a 端口（1a 和 2a）。
.. 将主机 2 路径 A（蓝色）HBA 端口连接到控制器 A 的 c 端口（1c 和 2c）。
.. 将主机 2 路径 B（红色）HBA 端口连接到控制器 B 的 c 端口（1c 和 2c）。
+

.Fabric-attached 拓扑
--
以下示例显示了使用光纤连接拓扑将存储系统连接到主机的布线情况。

.EF50 具有两个 4 端口 64Gb FC I/O 模块
[%collapsible]
=====
.关于此任务
* 布线示例显示了插槽 1 和 2 中的主机 I/O 模块。这是 EF50 存储系统支持的最大主机 I/O 模块数。但是，仅需要插槽 1 中的主机 I/O 模块；插槽 2 中的主机 I/O 模块是可选的。
+
如果您的存储系统安装了一个主机 I/O 模块，则可以忽略与其他主机 I/O 模块的缆线连接，而仅将缆线连接到已安装的主机 I/O 模块。

* 光纤连接存储系统具有两条独立的冗余交换机路径：交换机 1 路径和交换机 2 路径。
+
** 交换机 1 路径连接显示为主机和控制器上的蓝色电缆和蓝色端口。它通过交换机 1 将每个主机上的 HBA 端口连接到控制器 A 和控制器 B 的 a 和 c 端口。
** 交换机 2 路径连接显示为主机和控制器上的红色电缆和红色端口。它通过交换机 2 将每个主机上的 HBA 端口连接到控制器 A 和控制器 B 的 b 和 d 端口。




.步骤
. 将主机连接到交换机。
+
您可以使用交换机上的任何端口。

+
.. 将主机 1 和主机 2 交换机 1 路径（蓝色）HBA 端口连接到交换机 1。
.. 将主机 1 和主机 2 的交换机 2 路径（红色）HBA 端口连接到交换机 2。


. 将开关连接到控制器：
+
.. 电缆开关 1（蓝色）到控制器 A 的 a 和 c 端口（1a、2a、1c 和 2c）。
.. 将电缆开关 1（蓝色）连接到控制器 B 的 a 和 c 端口（1a、2a、1c 和 2c）。
.. 将电缆开关 2（红色）连接到控制器 A 的 b 和 d 端口（1b、2b、1d 和 2d）。
.. 将电缆开关 2（红色）连接到控制器 B 的 b 和 d 端口（1b、2b、1d 和 2d）。
+
